Backtracking (Quay Lui)

Kỹ thuật duyệt nghiệm cạn kiệt. Thử mọi ngã rẽ, đụng ngõ cụt thì quay lại (undo) và thử đường khác.
🔥 Insight: Backtracking = Chọn (Choose) ➔ Khám phá (Explore) ➔ Hoàn tác (Un-choose)

Giải ngố Backtracking cho người mới

Ví dụ: Lạc vào Mê Cung

Hãy tưởng tượng bạn đang ở trong một mê cung có rất nhiều ngã ba. Bạn làm sao để thoát ra?

  • Bạn chọn đi vào Ngã rẽ Trái. Bạn đánh dấu đường này là "đang đi".
  • Đi được một đoạn, bạn đụng bức tường (Ngõ cụt / Vi phạm điều kiện).
  • Bạn không thể đi tiếp. Bạn bước lùi lại (Backtrack) về cái ngã ba ban đầu, xóa bỏ vết chân ở ngã rẽ Trái (để dọn đường cho việc khác).
  • Sau đó bạn thử rẽ sang Ngã rẽ Phải.

Backtracking trong code chính xác là như vậy: Nó dùng đệ quy để đi sâu vào 1 nhánh. Nếu sai, nó HỦY BỎ (Undo) trạng thái vừa thêm vào để khôi phục lại mảng/biến như cũ, rồi chạy tiếp vòng lặp sang nhánh khác.

Template Code Thần Thánh

Hầu như 99% các bài Backtracking đều viết theo cấu trúc này: 

function backtrack(state) {
  // 1. Điều kiện dừng (Lưu kết quả)
  if (isSolution(state)) {
    results.push([...state]);
    return;
  }

  // 2. Duyệt các ngã rẽ có thể đi
  for (let choice of choices) {
    if (isValid(choice)) {
      state.push(choice);     // CHOOSE (Chọn)
      
      backtrack(state);       // EXPLORE (Đệ quy đi sâu)
      
      state.pop();            // UN-CHOOSE (Quay lui!!!)
    }
  }
}

Nâng cao: Cắt Tỉa (Pruning)

Điểm khác biệt giữa Backtracking và Brute Force (Vét cạn ngốc nghếch) là Pruning.

Bằng hàm isValid(), ta có thể dự đoán sớm một con đường là sai lầm và cắt bỏ nguyên cả nhánh đó mà không thèm đệ quy xuống nữa (Ví dụ: Trong Sudoku, nếu điền số 5 bị trùng hàng, ta cắt ngay lập tức nhánh đó).

Dấu hiệu nhận biết

Đề bài bắt tìm: "Tất cả các tổ hợp...", "Tất cả các hoán vị...", "Liệt kê mọi đường đi...". Kích thước input thường rất nhỏ (N < 20) do độ phức tạp O(2ⁿ) hoặc O(N!).
