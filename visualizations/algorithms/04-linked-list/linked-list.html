Linked List (Danh Sách Liên Kết)

Cấu trúc dữ liệu linh hoạt, khắc phục nhược điểm của Mảng. Yêu cầu kỹ năng thao tác Con trỏ (Pointers).

Căn bản về Linked List

Cấu trúc 1 Node cơ bản
Head ➔
Data (Giá trị)
.next (Con trỏ)
NULL

Tại sao dùng Linked List?

Khác với Mảng (Array) cần cấp phát ô nhớ liên tục, Linked List tạo các ô nhớ rải rác trong RAM và xâu chuỗi chúng lại.

Thao tác Array (Mảng) Linked List
Kích thước Cố định / Khó co giãn Linh hoạt vô hạn
Chèn/Xóa (Đầu mảng) O(n) - Phải dịch toàn bộ O(1) - Rất nhanh
Truy cập ngẫu nhiên (arr[i]) O(1) - Cực nhanh O(n) - Phải lội từ Head

3 Quy tắc Vàng khi Code

  • 1 Tránh Null Reference: Luôn kiểm tra if (!head) đầu tiên. Khi dùng curr.next, hãy chắc chắn curr không bị Null.
  • 2 Nguyên tắc "Giữ dây": Khi muốn bẻ mũi tên của Node A sang chỗ khác, phải gán Node B vào một biến temp trước kẻo toàn bộ đuôi phía sau bị rơi vào hố đen (Memory Leak).
  • 3 Dùng Dummy Node: Nếu thuật toán có khả năng làm mất hoặc thay đổi Node Head gốc (như trộn mảng, xóa Node đầu), hãy tạo một Node giả `ListNode(-1)` cắm vào trước Head để làm mỏ neo.
