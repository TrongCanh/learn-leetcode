## 📖 Lý thuyết

### 🔍 1. AOF là gì?

**AOF (Append-Only File) = Log tất cả write commands. Replay log để khôi phục data.**

```
┌──────────────────────────────────────────────────────────────┐
│                    AOF — APPEND-ONLY FILE                           │
│                                                               │
│  Write operation:                                              │
│  SET user:1 "Alice"                                           │
│       │                                                        │
│       ▼                                                        │
│  ┌──────────────────────────────────────────────────────┐    │
│  │                   appendonly.aof                       │    │
│  │                                                        │    │
│  │  *2                                         │         │    │
│  │  $3                                         │         │    │
│  │  SET                                        │         │    │
│  │  $6                                         │         │    │
│  │  user:1                                     │         │    │
│  │  $5                                         │         │    │
│  │  Alice                                      │         │    │
│  │                                                        │    │
│  └──────────────────────────────────────────────────────┘    │
│       │                                                        │
│       │ Persist to disk                                       │
│       ▼                                                        │
│  Disk: appendonly.aof (growing file)                          │
│                                                               │
│  Restore: Replay commands from start                            │
│                                                               │
└──────────────────────────────────────────────────────────────┘
```

### 🔍 2. RDB vs AOF

```
┌─────────────────────────────────────────────────────────────┐
│              RDB vs AOF                                         │
├─────────────────────────────────────────────────────────────┤
│                                                               │
│  RDB:                                                        │
│  ├── Point-in-time snapshot                                 │
│  ├── Compact binary file                                   │
│  ├── Fast restore (full dataset)                           │
│  ├── Good for backups                                      │
│  └── Data có thể mất (giữa 2 snapshots)                   │
│                                                               │
│  AOF:                                                        │
│  ├── Append-only log (write every operation)                │
│  ├── More durable (tunable fsync)                          │
│  ├── Can lose only 1 second of data (fsync everysec)       │
│  ├── File grow larger (rewrite for compaction)              │
│  └── Slower restore (replay all commands)                   │
│                                                               │
└─────────────────────────────────────────────────────────────┘
```

### 🔍 3. fsync Policies

```
fsync = force write to disk (SYNCHRONIZE)

┌─────────────────────────────────────────────────────────────┐
│              FSYNC POLICIES                                   │
├─────────────────────────────────────────────────────────────┤
│                                                               │
│  appendfsync always:                                        │
│  ├── fsync() sau MỖI operation                             │
│  ├── 0 data loss                                          │
│  ├── ⚠️ Rất chậm (I/O bound)                            │
│  └── ⚠️ Không nên dùng trong production                  │
│                                                               │
│  appendfsync everysec (RECOMMENDED):                        │
│  ├── fsync() mỗi GIÂY                                    │
│  ├── Max 1 giây data loss                                 │
│  ├── ✅ Good balance: speed + durability                  │
│  └── ✅ Most production deployments                        │
│                                                               │
│  appendfsync no:                                           │
│  ├── OS quyết định khi nào fsync                         │
│  ├── Fastest                                                 │
│  ├── ⚠️ Có thể mất nhiều data                          │
│  └── ⚠️ Kernel crash = all AOF data lost                │
│                                                               │
└─────────────────────────────────────────────────────────────┘
```

### 🔍 4. AOF Rewrite (Compaction)

```
Problem: AOF file grow vô hạn!
  SET counter 1    → *3\r\n$3\r\nSET\r\n$7\r\ncounter\r\n$1\r\n1\r\n
  INCR counter     → *2\r\n$4\r\nINCR\r\n$7\r\ncounter\r\n
  INCR counter     → *2\r\n$4\r\nINCR\r\n$7\r\ncounter\r\n
  ... → 1 triệu INCRs = 1 triệu lines!

Solution: AOF Rewrite (BGREWRITEAOF)
  Tạo file mới chỉ chứa current state:
  SET counter 1000000

┌─────────────────────────────────────────────────────────────┐
│  BGREWRITEAOF Process:                                  │
│    1. Redis fork child process                           │
│    2. Child đọc current state → write to new AOF       │
│    3. Parent tiếp tục append vào AOF cũ              │
│    4. Khi child xong → replace AOF cũ bằng AOF mới   │
│                                                               │
│  Config:                                                   │
│    auto-aof-rewrite-percentage 100                       │
│    auto-aof-rewrite-min-size 64mb                       │
│    → Rewrite khi AOF > 100% last rewrite AND > 64MB    │
└─────────────────────────────────────────────────────────────┘
```

### 🔍 5. Redis 7 Multi-Part AOF

```
Redis 7+: AOF = base file + incremental chunks

appendonly.aof (base) + appendonly.001.incr.aof (incremental)

┌─────────────────────────────────────────────────────────────┐
│  Redis 7 AOF structure:                                   │
│                                                               │
│  appendonly.aof.base.12345.rdb  ← Full snapshot          │
│  appendonly.aof.incr.00001.asc  ← Incremental log     │
│  appendonly.aof.incr.00002.asc  ← Incremental log     │
│                                                               │
│  → Fast startup (small base + few inc files)           │
│  → Efficient storage (incremental writes)                 │
└─────────────────────────────────────────────────────────────┘
```

## 🧠 Phân tích & Hướng tư tư

### 🤔 Khi nào dùng gì?

```
┌─────────────────────────────────────────────────────────────┐
│  Setup recommendations:                                        │
├─────────────────────────────────────────────────────────────┤
│                                                               │
│  No persistence (cache only):                                 │
│    → Không dùng RDB hoặc AOF                              │
│    → Data có thể mất bất kỳ lúc nào                      │
│                                                               │
│  RDB only:                                                   │
│    → Backup database periodically                            │
│    → Can tolerate losing few hours of data                  │
│    → Large datasets, write-heavy                            │
│                                                               │
│  AOF only (everysec):                                       │
│    → Most common for production                             │
│    → Good durability with decent performance                │
│                                                               │
│  RDB + AOF (hybrid):                                        │
│    → Best of both worlds                                    │
│    → AOF = latest commands                                 │
│    → RDB = fast restore base                              │
│    → Recommended for critical data                          │
│                                                               │
└─────────────────────────────────────────────────────────────┘
```

### ⚠️ Common Pitfalls

| Sai lầm | Giải thích |
|---------|-----------|
| **fsync always → performance chậm** | fsync mỗi write = I/O bound → Redis throughput giảm đáng kể |
| **AOF file quá lớn** | AOF rewrite tốn resource. Dùng `aof-use-rdb-preamble yes` (Redis 7+) |
| **Dùng AOF no fsync** | Kernel crash → tất cả AOF data mất. Chỉ dùng cho testing |

### 🔑 Key Insight

> **AOF = Append-only log = durable. `everysec` = ngàn 1 giây data loss. Rewrite = compaction để tránh file grow vô hạn. Hybrid (RDB+AOF) = best of both worlds.**

## ✅ Ví dụ

```python
import redis

r = redis.Redis(host='localhost', port=6379, db=0)

def check_persistence_config():
    """Kiểm tra persistence configuration"""
    info = r.info('persistence')
    return {
        "aof_enabled": info.get('aof_enabled'),
        "aof_current_size": info.get('aof_current_size'),
        "aof_base_size": info.get('aof_base_size'),
        "aof_pending_rewrite": info.get('aof_pending_rewrite'),
        "aof_last_write_status": info.get('aof_last_write_status'),
        "aof_last_fsync_status": info.get('aof_last_fsync_status'),
        "rdb_save_in_progress": info.get('rdb_save_in_progress'),
    }

def trigger_aof_rewrite():
    """Trigger AOF rewrite (background)"""
    result = r.bgrewriteaof()
    print(f"BGREWRITEAOF triggered: {result}")
    return result

def check_aof_size():
    """Theo dõi AOF size growth"""
    info = r.info('persistence')
    aof_size = info.get('aof_current_size', 0)
    base_size = info.get('aof_base_size', 0)

    print(f"AOF size: {aof_size / (1024*1024):.2f} MB")
    print(f"AOF base: {base_size / (1024*1024):.2f} MB")
    print(f"Growth: {(aof_size - base_size) / (1024*1024):.2f} MB")

def get_persistence_stats():
    """Stats về persistence operations"""
    info = r.info('persistence')
    stats = {
        'aof_last_write_status': info.get('aof_last_write_status'),
        'aof_last_fsync_status': info.get('aof_last_fsync_status'),
        'rdb_last_save_time': info.get('rdb_last_save_time'),
        'rdb_last_bgsave_status': info.get('rdb_last_bgsave_status'),
        'aof_rewrite_scheduled': info.get('aof_rewrite_scheduled'),
    }
    return stats

# ─── Redis Config (redis.conf) ───
REDIS_CONF_AOF = """
# AOF Configuration

# Enable AOF
appendonly yes

# AOF filename
appendfilename "appendonly.aof"

# fsync policy
# always = every write (slow)
# everysec = every second (recommended)
# no = let OS decide
appendfsync everysec

# Rewrite when AOF grows by 100%
auto-aof-rewrite-percentage 100

# Minimum AOF size before rewrite (64MB)
auto-aof-rewrite-min-size 64mb

# Continue rewrite even if AOF is truncated
aof-load-truncated yes

# Enable RDB preamble (Redis 7+)
aof-use-rdb-preamble yes
"""

# ─── Demo ───
if __name__ == "__main__":
    print("=== AOF Stats ===")
    stats = check_persistence_config()
    for k, v in stats.items():
        print(f"  {k}: {v}")

    print("\n=== AOF Size ===")
    check_aof_size()

    print("\n=== Persistence Stats ===")
    pstats = get_persistence_stats()
    for k, v in pstats.items():
        print(f"  {k}: {v}")

    print("\n=== Trigger AOF Rewrite ===")
    trigger_aof_rewrite()
```
