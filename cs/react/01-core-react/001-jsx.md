# JSX — Thực Chất Là Gì, Tại Sao Cần Nó

## Câu hỏi mở đầu

Bạn viết dòng này mỗi ngày:

```jsx
const element = <h1 className="title">Hello, {name}!</h1>;
```

Nhìn giống HTML. Trình duyệt chạy JavaScript. Vậy:

- Trình duyệt hiểu cú pháp `<h1>` không? **Không.**
- Babel/Webpack transform gì? Và transform thành cái gì?
- Tại sao phải dùng `className` thay vì `class`?
- `{name}` bên trong hoạt động ra sao?
- JSX có phải là HTML không? Có phải là string không?

Nếu bạn không trả lời được tất cả, bài này dành cho bạn.

---

## 1. JSX Compile Thành Gì?

### Không Phải HTML. Không Phải String.

JSX là **syntactic sugar** — một lớp cú pháp viết tắt cho `React.createElement()`.

Babel (hoặc TypeScript compiler) transform JSX thành function calls khi build.

```jsx
// Code bạn viết
const element = <h1 className="title">Hello, {name}!</h1>;

// ↓ Babel transform thành:
const element = React.createElement(
  'h1',                      // tag name (string)
  { className: 'title' },    // props object
  'Hello, ', name, '!'      // children (string hoặc element)
);

// ↓ React.createElement trả về plain object (Virtual DOM node):
{
  type: 'h1',
  key: null,
  ref: null,
  props: {
    className: 'title',
    children: ['Hello, ', 'Alice', '!']
  }
}
```

**Điểm then chốt**: JSX trả về **plain JavaScript object** — không phải HTML string, không phải DOM element. React sau đó so sánh object này với object cũ (diffing), rồi chỉ cập nhật phần DOM thực sự thay đổi.

### Nhiều Cấp Lồng Nhau

```jsx
// JSX bạn viết
const list = (
  <ul>
    <li>Item 1</li>
    <li>Item 2</li>
    <li>Item 3</li>
  </ul>
);

// Compile thành nested createElement calls
React.createElement('ul', null,
  React.createElement('li', null, 'Item 1'),
  React.createElement('li', null, 'Item 2'),
  React.createElement('li', null, 'Item 3')
);
```

### Component là Function

```jsx
// Component viết hoa — được truyền làm argument
const element = <Welcome name="Alice" />;

// ↓ Compile thành:
React.createElement(Welcome, { name: 'Alice' });
//              ↑
//       Function reference, NOT string
```

## 2. Tại Sao Dùng `className` Thay Vì `class`?

Vì `class` là **reserved keyword** trong JavaScript:

```javascript
// ❌ SyntaxError: Unexpected token
const x = <h1 class="title">Hello</h1>;

// ✅ Hợp lệ
const x = <h1 className="title">Hello</h1>;
```

Tương tự với các attribute khác:

| HTML attribute | JSX property | Lý do |
|---|---|---|
| `class` | `className` | `class` là keyword trong JS |
| `for` | `htmlFor` | `for` là keyword trong JS |
| `tabindex` | `tabIndex` | camelCase convention |
| `stroke-width` | `strokeWidth` | camelCase convention |
| `onclick` | `onClick` | camelCase convention |
| `onmouseenter` | `onMouseEnter` | camelCase convention |

**Pattern**: JSX attributes tuân theo **camelCase** JavaScript convention, không phải kebab-case HTML.

## 3. `{expression}` — Nhúng JavaScript Trong JSX

```jsx
const name = 'Alice';
const age = 30;
const isActive = true;

// Variable interpolation
const greeting = <h1>Hello, {name}!</h1>;

// Expression evaluation
const double = <p>Double 5 = {5 * 2}</p>;        // → 10
const greeting2 = <p>{isActive ? 'Active' : 'Inactive'}</p>;

// Function call
const getMessage = () => 'Message from function';
const msg = <p>{getMessage()}</p>;

// Array (React flatten children)
const items = [1, 2, 3];
const list = <ul>{items.map(x => <li key={x}>{x}</li>)}</ul>;
```

### Expression vs Statement

```jsx
// ✅ Expression — tính được, dùng được trong {}
5 + 3                    // → 8
isLoggedIn ? 'Hi' : 'Login'  // → string
users.map(u => u.name)   // → array
count + 1               // → number

// ❌ Statement — không tính được, KHÔNG dùng được trong {}
if (x) { doSomething() }    // ❌
for (let i = 0; i < 3; i++) // ❌
switch (x) { case 1: ... }  // ❌
```

### `&&` và `||` Trong JSX — Có Caveat

```jsx
// ✅ Boolean short-circuit — phổ biến
{isLoggedIn && <Dashboard />}  // isLoggedIn=true → Dashboard
                              // isLoggedIn=false → nothing rendered

// ⚠️ CAVEAT: 0 và '' là falsy nhưng React VẪN render!
{items.length && <List items={items} />}
// Khi items.length = 0 → React render ra "0" (số 0)!
{items.length > 0 && <List items={items} />}  // ✅ Fix
{isEmpty ? null : <List items={items} />}     // ✅ Fix

// ✅ OR fallback
{user.name || 'Anonymous'}  // render "Anonymous" nếu name falsy
```

## 4. JSX Không Phải HTML — Sự Khác Biệt Quan Trọng

### Khác Nhau Về Props

```jsx
// HTML: attributes lowercase, có thể viết tắt
<input type="text" disabled />
<input type="text" disabled="disabled" />
<input type="text" disabled={true} />

// JSX: camelCase, boolean phải explicit
<input type="text" disabled />
<input type="text" disabled={true} />

// ⚠️ HTML: disabled="" → disabled=true
// ⚠️ JSX: disabled={0} → KHÔNG disabled (vì 0 là falsy!)
```

### Style Prop — Object, Không String

```jsx
// ❌ HTML style là string
<div style="color: red; font-size: 16px"></div>

// ✅ JSX style là JavaScript object (camelCase)
<div style={{ color: 'red', fontSize: '16px' }}></div>

// Multiple styles
const cardStyle = {
  backgroundColor: '#fff',
  borderRadius: '8px',
  padding: '16px',
  marginTop: '8px',
};
<div style={cardStyle}>Content</div>
```

### Self-Closing Tags

```jsx
// HTML: có thể không có trailing slash
<input type="text" name="email">

// JSX: PHẢI có trailing slash
<input type="text" name="email" />
<br />
<img src="logo.png" alt="Logo" />
```

## 5. Virtual DOM Node — Bản Chất Của Mọi Thứ

Sau khi JSX compile, kết quả là **plain JavaScript object**:

```javascript
// Virtual DOM node = plain object
{
  $$typeof: Symbol.for('react.element'),

  type: 'h1',
  key: null,
  ref: null,

  props: {
    className: 'title',
    children: [
      'Hello, ',
      'Alice',  // value từ biến
      '!'       // string literal
    ]
  },

  // Internal fields
  _owner: null,
  _store: { validated: false }
}
```

**Đây là nền tảng cho mọi thứ React làm**:

```
Component returns JSX
  ↓
JSX = React.createElement() call
  ↓
Virtual DOM Node = Plain Object
  ↓
React builds VDOM tree (in-memory)
  ↓
React diffs old tree vs new tree (Reconciliation)
  ↓
React computes minimal DOM operations
  ↓
React applies only changed parts to Real DOM
```

## 6. `key` Prop — Tại Sao Quan Trọng

### Key Giúp React Xác Định Element Nào Thay Đổi

```jsx
const items = [{ id: 1, name: 'Alice' }, { id: 2, name: 'Bob' }];

// ✅ Dùng stable unique key
items.map(item => <li key={item.id}>{item.name}</li>);

// ❌ KHÔNG dùng index (trừ khi list không thay đổi)
items.map((item, index) => <li key={index}>{item.name}</li>);
```

### Demo Bug Khi Dùng Index

```jsx
function BuggyList() {
  const [items, setItems] = useState(['Alice', 'Bob', 'Carol']);

  // Dùng index làm key — CÓ BUG!
  return (
    <ul>
      {items.map((name, i) => (
        <li key={i}>
          <input type="text" placeholder={`Input for ${name}`} />
          {name}
        </li>
      ))}
    </ul>
  );
}

function prependItem() {
  setItems(['Zara', ...items]);
  // Kết quả KHÔNG đúng:
  // <li key=0>: Input [giá trị cũ của Bob] + Zara
  // <li key=1>: Input [giá trị cũ của Carol] + Bob
  // <li key=2>: Input [giá trị cũ của Alice] + Carol
  // → Input values bị scramble!
}
```

**Tại sao bug xảy ra?**

```
TRƯỚC KHI THÊM:
  key=0: <li>Alice</li>        <input placeholder="Input for Alice" />
  key=1: <li>Bob</li>          <input placeholder="Input for Bob" />
  key=2: <li>Carol</li>        <input placeholder="Input for Carol" />

SAU KHI PREPEND ['Zara']:
  key=0: <li>Zara</li>        <input placeholder="Input for Alice" />
  key=1: <li>Alice</li>        <input placeholder="Input for Bob" />
  key=2: <li>Bob</li>          <input placeholder="Input for Carol" />
  key=3: <li>Carol</li>        <input [mới]

→ React: key=0 đã có (Alice), giờ là Zara → UPDATE content
→ React giữ nguyên input DOM node (vì key match)
→ Result: Input của Alice hiển thị, nhưng label là Zara!
```

### Đúng Cách

```jsx
// ✅ Dùng stable ID
items.map(item => <li key={item.id}>{item.name}</li>);

// ✅ Dùng index khi list STATIC (không thay đổi)
const staticItems = ['Home', 'About', 'Contact'];
staticItems.map((item, i) => <li key={i}>{item}</li>); // OK
```

## 7. Liên Hệ Với JavaScript Core

### JSX Compile Thành Function Calls

```javascript
// JSX transform = syntactic transformation
// Đây là JavaScript, sau transform:

// <div>Hello</div>
React.createElement('div', null, 'Hello');

// <div id="app">Content</div>
React.createElement('div', { id: 'app' }, 'Content');

// <Component prop={value} />
React.createElement(Component, { prop: value });
```

### Props = Arguments Trong Function Call

```jsx
<UserCard name="Alice" age={30} active={true} />

// = compile thành:
React.createElement(UserCard, {
  name: 'Alice',
  age: 30,
  active: true,
  children: undefined
});
```

### Children Là Arguments

```jsx
<Card>
  <Title>Hello</Title>
  <Body>Content here</Body>
</Card>

// = compile thành:
React.createElement(Card, null,
  React.createElement(Title, null, 'Hello'),
  React.createElement(Body, null, 'Content here')
);

// props.children = [TitleElement, BodyElement]
```

## 8. Các Traps Phổ Biến

### ❌ Trap 1: Nhầm JSX Với HTML String

```jsx
// ❌ SAI — JSX KHÔNG PHẢI STRING
const element = "<h1>Hello</h1>";
document.body.innerHTML = element; // Works nhưng KHÔNG phải React!

// ✅ ĐÚNG — JSX là React element object
const element = <h1>Hello</h1>;
ReactDOM.createRoot(root).render(element);
```

### ❌ Trap 2: Return Nhiều Elements Không Wrap

```jsx
// ❌ SyntaxError: JSX fragments must be wrapped
function BadComponent() {
  return (
    <h1>Title</h1>
    <p>Description</p>
  );
}

// ✅ ĐÚNG: wrap trong div hoặc Fragment
function GoodComponent() {
  return (
    <div>
      <h1>Title</h1>
      <p>Description</p>
    </div>
  );
}

// ✅ HOẶC: dùng Fragment
function GoodComponent() {
  return (
    <>
      <h1>Title</h1>
      <p>Description</p>
    </>
  );
}
```

### ❌ Trap 3: `&&` Với Giá Trị 0

```jsx
// ❌ Bug: 0 là falsy nhưng React render nó!
const messages = [];

// Nếu bạn viết:
{ messages.length && <p>You have messages</p> }
// → Render ra "0" (số 0)!

// ✅ Fix bằng ternary
{ messages.length > 0 && <p>You have messages</p> }

// ✅ Hoặc explicit null
{ messages.length ? <p>You have messages</p> : null }
```

### ❌ Trap 4: Nhầm Boolean Attributes

```jsx
// ❌ SAI — disabled={false} KHÔNG disable input
<input disabled={false} /> // → KHÔNG disabled (sai logic!)

// ⚠️ HTML behavior: <input disabled=""> → disabled
// ⚠️ JSX behavior: <input disabled={false} /> → KHÔNG disabled
// → Nhưng <input disabled /> (không value) → disabled=true!

// ✅ Luôn dùng explicit boolean
<input disabled={true} />
<input disabled={false} />
```

## 9. Tổng Hợp

```
┌──────────────────────────────────────────────────────────────┐
│  JSX = Syntactic Sugar for React.createElement()              │
│                                                               │
│  <tag attr="val">{expression}</tag>                          │
│       ↓                                                        │
│  React.createElement('tag', { attr: 'val' }, expression)       │
│       ↓                                                        │
│  { type: 'tag', props: { attr: 'val', children: ... } }      │
│       ↓                                                        │
│  Plain Object (Virtual DOM Node)                               │
│       ↓                                                        │
│  React builds VDOM tree → Diff → Minimal DOM ops            │
│                                                               │
│  KEY DIFFERENCES FROM HTML:                                   │
│  ├── class → className                                        │
│  ├── for → htmlFor                                           │
│  ├── style="string" → style={{ object }}                   │
│  ├── self-closing tags cần />                                │
│  └── boolean props cần explicit {}                          │
│                                                               │
│  KEY PROPS:                                                   │
│  └── key: stable, unique in sibling list                     │
│                                                               │
│  EXPRESSIONS:                                                 │
│  ├── {} = any JS expression                                  │
│  ├── No if/for/switch statements                             │
│  └── && / || : watch out for falsy values (0, '')           │
└──────────────────────────────────────────────────────────────┘
```

## 10. Câu Hỏi Phỏng Vấn

### Q1: JSX compile thành gì?

**Trả lời:** JSX compile thành `React.createElement()` function calls, trả về plain JavaScript object (Virtual DOM node). Object có shape: `{ type, props: { children, ... }, key, ref }`. Babel transform JSX syntax thành `React.createElement()` calls trong quá trình build. Không có runtime JSX transform trong production — tất cả transform xảy ra lúc build.

### Q2: JSX khác HTML như thế nào?

**Trả lời:** JSX khác HTML ở nhiều điểm quan trọng: (1) `class` → `className`, `for` → `htmlFor`; (2) `style` nhận JavaScript object với camelCase, không phải CSS string; (3) Tất cả props phải camelCase; (4) Self-closing tags phải có `/>`; (5) JSX không phải string — compile thành function calls; (6) `disabled={false}` hoạt động khác với HTML attribute.

### Q3: Virtual DOM node là gì?

**Trả lời:** Virtual DOM node là plain JavaScript object mô tả UI element. Object có `type` (tag hoặc component), `props` (attributes và children), `key`, `ref`. React không trực tiếp tạo DOM — tạo object mô tả DOM mong muốn, so sánh với object cũ, rồi chỉ apply phần thay đổi. Đây là lý do VDOM diffing nhanh: chỉ so sánh objects, không touch real DOM.

### Q4: Tại sao cần `key` prop trong list?

**Trả lời:** `key` giúp React xác định element nào thay đổi khi list re-render. Không có key, React dùng index và không thể distinguish giữa item being moved, added, hoặc removed. Dùng index có thể gây bugs khi list thay đổi — input values bị scramble, state bị mất. Key phải **stable** (không thay đổi across renders) và **unique trong sibling list**.

### Q5: Expression và statement khác nhau thế nào trong JSX?

**Trả lời:** Expression = anything evaluates to a value (variable, function call, ternary, arithmetic). Dùng được trong `{}`. Statement = imperatives (if, for, switch). Không dùng được trong JSX vì JSX là expression-oriented. Thay thế `if` bằng ternary `condition ? true : false`, thay thế `for` bằng `array.map()`.
